Zmq，PyZmq笔记

ZMQ(以下ZeroMQ简称ZMQ)是一个简单好用的传输层，像框架一样的一个socket library，他使得Socket编程更加简单、简洁和性能更高。是一个消息处理队列库，可在多个线程、内核和主机盒之间弹性伸缩。ZMQ的明确目标是“成为标准网络协议栈的一部分，之后进入Linux内核”。现在还未看到它们的成功。但是，它无疑是极具前景的、并且是人们更加需要的“传统”BSD套接字之上的一 层封装。ZMQ让编写高性能网络应用程序极为简单和有趣。
zmq是一个消息队列。可以在进程内、进程间、TCP、多播中，以消息为单位传输数据，而不是socket的字节流。

ZMQ三种基本通讯模式

  1. 应答模式(REQ-REP)：
    消息是双向的，REQ-REP套接字发送和接受消息必须是成对的。
    客户端必须先发送消息，才能接收消息，如果打乱了这个顺序（如连续发送两次）则会报错。类似地，服务端必须先进行接收，后进行发送。
  2. 订阅发布模式(PUB-SUB)
    消息单向的，服务端只负责发送数据，不能接收数据，且不保证把全部的信息都发送给订阅者。
    PUB-SUB套接字组合是异步的。客户端只能接收消息，如果向PUB套接字发送消息则会报错；类似地，服务端可以不断地使用send ()发送消息，但不能再PUB套接字上使用recv ()。
    关于PUB-SUB套接字，还有一点需要注意：你无法得知SUB是何时开始接收消息的。就算你先打开了SUB套接字，后打开PUB发送消息，这时SUB还是会丢失一些消息的，因为建立连接是需要一些时间的。很少，但并不是零。解决此问题需要在PUB端加入sleep。

依赖

  • Python

安装

  1. 下载zmq源码，下载地址

  2. 安装过程中会需要libtool, pkg-config, build-essential, autoconf和automake，先行安装

    sudo apt-get install libtool, pkg-config, build-essential, autoconf,automake

  3. 进入源码目录，进行安装

    cd zeromq-3.2.5
./configure
make && make check
sudo make install

  4. 安装pyZmq，使用pip直接安装
    考虑过版本问题，官方表示
    PyZMQ ≥ 2.2.0 fully supports the 3.x and 4.x APIs of libzmq，2.2以上版本是完全支持Zmq3.x和4.x的

    pip install pyzmq
