Win32com笔记

Pywin32 是python windows的扩展，不跨平台，通过COM口进行操作，几乎封装了所有python下面的win32相关的操作，例如win32api,win32gui等等。笔者用来对Word进行操作，其可以打开Word应用，新建文档，插入，另存，设置样式，查找，替换，页眉页脚，表格等操作，只对用到的功能进行简单总结。

使用

基本命令

  • 启动Word应用

    #如果存在已启动的word应用，则使用已打开的应用
w = win32com.client.Dispatch('Word.Application')
#使用启动独立的进程，始终新建一个应用
w = win32com.client.DispatchEx('Word.Application')

  • 对应用状态进行设置

    # 后台运行，不显示，不警告
w.Visible = False
w.DisplayAlerts = False

  • 使用应用打开文件

    #打开新的文件
doc = w.Documents.Open( FileName = filenamein )
#创建一个新文档
doc = w.Documents.Add()

  • 获取文档区域，并进行操作

    #获得文档开头
myRange = doc.Range(0,0)
#插入文字
myRange.InsertBefore('Hello from Python!')

  • 文档保存

    #保存为当前文档
doc.Save()
#文档另存为
doc.SaveAs('test.doc')

  • 关闭文档

    doc.Close()

  • 退出应用

    w.Quit()

    应用

创建Word文档Demo

import win32com
from win32com.client import Dispatch, constants

#使用启动独立的进程
w = win32com.client.DispatchEx('Word.Application')
# 后台运行，不显示，警告
w.Visible = False
w.DisplayAlerts = True

# 创建新的文档
doc = w.Documents.Add()

# 插入文字
myRange = doc.Range(0,0)
myRange.InsertBefore('Hello from Python!')

#运行时将文档保存到了'文档'文件夹下
doc.SaveAs('test.doc')

# 关闭
doc.Close()
w.Quit()

错误汇总

  • 在注册表中找不到Python
    Python Version 3.4 required which was not found in the registry
    原因：windows平台下，32/64bit第三方库会分别在32/64bit的注册表中查找python路径，而安装的python 64bit只会把安装信息写入32bit的注册表中，当安装64bit的win32com时，在注册表查找不到python的安装信息。

    Python的一些第三方库只到注册表的HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下寻找Python。但是装好的64位Python在HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下建立Python的信息。所以，在cmd输入regedit然后将HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下的值，复制到HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下，当然没有路径的话要新建。

    解决办法：添加相应的python安装信息到注册表，可以手动新建也可以修改date文件夹下的reg文件来一键导入。

  • import win32com出错
    importError：DLL load failed %1 不是有效的Win32应用程序
    原因：win32com的版本位数与python版本位数不对应
    解决办法：cmd执行python，查看python位数，下载对应安装包，再次安装

  • 同时安装WPS和Office时，使用word.application创建文档时，打开的是WPS，且保存时会出错
    笔者卸载了WPS，暂未得到解决

注意

  • Windows下使用相对路径保存文件，相对于当前用户的文档目录
  • 注册组件的API可以通过OLEView查看参考
