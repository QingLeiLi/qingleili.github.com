Hexo笔记

安装

依赖

  • NodeJs，Npm

安装

  1. 使用npm直接安装
    npm install hexo -g

其他

  • 升级到最新版本
    npm update hexo -g

使用

基本命令

  • 初始化目录，省略工程名则初始化当前目录
    hexo init [folder]

  • 创建博客文章
    hexo new "Hello World"

  • 生成网站
    hexo generate

  • 运行Hexo服务，默认地址为http://localhost:4000
    hexo server

  • 部署网站
    hexo deploy

应用

使用Hexo创建博客

  1. 初始化目录，并切到博客目录 

    hexo init blog
cd blog

  2. 安装项目依赖包 

    npm install

  3. 生成静态网页 

    hexo generate

  4. 运行项目，查看博客效果，默认地址http://localhost:4000 

    hexo server

部署博客到GitPage上

  1. 登陆GitHub创建仓库(因GitHub限制，仓库名必须为用户名.github.com)

  2. 编辑项目目录下_config.yml配置文件，找到deploy配置项并修改

    deploy:
  type: git  #仓库类型
  repository: git@github.com:用户名/用户名.github.com.git  #仓库地址
  branch: master  #仓库分支

    注意：Hexo 3.0之前的版本，需要修改type:github，并略过3操作

  3. 安装deploy git

    npm install hexo-deployer-git --save

    注意：在Hexo 3.0版本后deploy git被分出来了，需要单独安装，之前的版本无须安装。

  4. 将项目部署到GitHub上 

    hexo deploy
  5. 访问http://用户名.github.io/就应该可以看到博客了。

错误汇总

  • 部署网站时找不到github：Deployer not found: github
    解决办法：见部署博客到GitPage上第2，3条

  • 部署时执行hexo deploy命令行没有任何输出，也没有错误提示
    解决办法：在_config.yml文件中，找到deploy:标签，确保每个冒号（type，repository，branch等）后面有空格，否则就会出现上述问题。

  • 访问网页时，文章不显示，后台报错
    ERROR Process failed: _posts/git.md YAMLException: can not read a block mapping entry; a multiline key may not be animplicit key at line 4, column 1
    原因：yaml配置文件对格式要求很严格，文章的头部信息，每一个键后必须有一个空格，否则会报错。
    解决办法：检查文章头部信息格式将

    tags:git

    改为

    tags: git

    注意空格区别。
