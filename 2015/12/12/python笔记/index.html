安装

  1. Ubuntu

    • 安装Python3 

      #安装python3主体
sudo apt-get install python3
#检查是否存在python3链接，没有则新建
sudo ln -s /usr/bin/python3.2 /usr/bin/python

其他

  1. Ubuntu

    • 安装pip 

      #安装pip主体
sudo apt-get install python-pip
#检查是否存在python3链接，没有则新建
sudo ln -s /usr/bin/python3.2 /usr/bin/python

  2. Windows

    • 安装MySQLdb
      Windows下有mysqldb的安装包，安装很简单。需要注意的是mysqldb只有32bit，所以python必须为32位版本，否则会报无效的DLL错误。而且可能会有注册表问题，详见错误汇总。

      1. 下载MySQLdbWindows安装包
      2. 安装即可

      3. 测试

        >import MySQLdb
>

错误汇总

  • pip安装第三方库时，找不到python.h

    src/MD2.c:31:20: fatal error: Python.h: No such file or directory compilation terminated.
    error: command ‘gcc’ failed with exit status 1

    解决办法：安装python开发包

    sudo apt-get install python-dev

  • Ubuntu apt-get install python-dev安装python-dev失败

    The following packages have unmet dependencies:
    python-dev : Depends: python2.7-dev (>= 2.7.3) but it is not going to be installed
    E: Unable to correct problems, you have held broken packages.

    解决办法：更新Ubuntu源

    echo "deb http://us.archive.ubuntu.com/ubuntu/ precise-updates main restricted" | sudo tee -a /etc/apt/sources.list.d/precise-updates.list
sudo apt-get update
sudo apt-get install python-dev

  • Pip安装scons报错

    error: option –single-version-externally-managed not recognized

    解决办法：增加–egg参数 

    sudo pip install --egg scons

  • Windows下在注册表中找不到Python

    Python Version 3.4 required which was not found in the registry

    原因：windows平台下，32/64bit第三方库会分别在32/64bit的注册表中查找python路径，而安装的python 64bit只会把安装信息写入32bit的注册表中，当安装64bit的win32com时，在注册表查找不到python的安装信息。

    Python的一些第三方库只到注册表的HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下寻找Python。但是装好的64位Python在HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下建立Python的信息。所以，在cmd输入regedit然后将HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下的值，复制到HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下，当然没有路径的话要新建。

    解决办法：添加相应的python安装信息到注册表，可以手动新建也可以修改date文件夹下的reg文件来一键导入。

  • Windows下import 出错不是有效的Win32应用程序

    importError：DLL load failed %1 不是有效的Win32应用程序

    原因：win32com的版本位数与python版本位数不对应
    解决办法：cmd执行python，查看python位数，下载对应安装包，再次安装

    注意

  • Python3已经不再支持mysqldb，使用pymysql或mysql官方的mysql.connector
