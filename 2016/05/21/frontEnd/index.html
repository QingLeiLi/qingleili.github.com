前端知识点

  1. HTML
    部分标签会经常使用，需要熟练掌握，知道其作用以及相关属性可以做到什么效果；对于不常用的标签要了解其使用效果，在具体场景中能够找到实现相应效果的标签即可。
    • 熟练掌握常见的html标签（div，form，input，a，table等等）的使用方法和相关属性；
    • 了解其他标签的作用和相关属性。
  2. CSS
    要求了解各个标签的css属性，熟悉css选择器的使用。
    • 熟练掌握css选择器的使用；
    • 掌握css选择器的权重设置，样式优先级；
    • 了解html标签的css属性
    • 熟练使用bootstrap框架
    • css的四种布局方式
  3. js
    js是当前前端开发中使用最多的，包括事件，数据处理，ajax等。
    • 熟练使用jquery框架
    • ajax异步获取数据
    • js事件绑定方法的异同（onclick，attach，addEventListener）
    • 事件的冒泡与捕获
    • 事件委托（用于管理某个区域内所有的事件）
    • 获取/设置html，css属性
  4. http协议
    web的访问是通过http协议完成的，必须对http协议有一定的了解。
    • 请求方式（GET，POST等）
    • cookie
    • session
    • http请求包
    • Http响应包
    • 状态码（404，500，302）
    • 缓存生存时间
  5. 安全
    要了解前端常见的攻击方式及防护手段。
    • csrf
    • xss
    • sql注入
    • 提交的表单要进行js验证，服务端再次验证
  6. git
    git是一个分布式的版本控制，项目开发的代码使用git进行代码合并。
    • clone
    • status
    • add
    • commit
    • pull
    • push
    • checkout
  7. 插件
    常用的插件有echarts和datatables。echarts是百度开发的一个图表库，支持的图表类型很多，可以使用其表现数据；datatables是jquery的一个表格插件，提供了分页，搜索，筛选等很多功能。两个插件都需要掌握使用。
    • echarts
    • datatables

前端开发流程

使用html与css开发出一整个网站是费时费力的，而且当前前端开发人员的能力也不够，我们所做的开发是基于模板的，更多的是后端程序的开发。

  1. 查找模板
    在各大模板网站查找模板(themeforest)查找比较好看，符合功能需求的模板，一般模板都会自带侧边框和用户，还有很多的插件示例，基本可以满足需求。
    模板挑选的标准就是可以直接拿来使用，形成大致的框架，以后的工作都是基于该模板的修改，要保证后续的修改不至于太麻烦。

  2. 下载模板并修正
    一般找到的模板都是收费的，有条件可以买；没有条件的话可以使用wget做整站克隆，将尽可能多的文件下载到本地，使网站可以本地运行，使用方法自行百度；

    模板一般会有一些cdn引用（jquery，bootstrap之类）或下载失败的资源（例如google字体库），需要手动将模板进行修改。

    1. 尽可能保证模板中不需要下载其他服务器资源
      因为网站的部署环境并不确定，在不允许访问外网的条件下使用其他站点资源可能造成排版或功能问题。

      使用文本编辑器打开整个模板文件夹，使用全局搜索http，将找到的内容下载到本地，并修改模板代码将源链接指向本地文件。

    2. 检查缺少的资源
      使用wget的情况下，部分资源可能没有进行下载，导致网页404错误，手动下载好了。

      打开模板网页，f12打开控制台，看是否有报错，404错误要查找原因，手动下载缺失文件，其他错误酌情处理。

      模板在使用前必须保证控制台不能有报错。

  3. Django创建项目
    django是python的一个mvc框架，可以去看django book，需要注意的是django版本在不断更新，书中一些配置与新版本不一致，学习的时候建议使用其指定的版本。

    创建django项目后，需要配置模板路径，数据库，静态文件等，具体配置自行百度。

    将模板的框架分离出来作为django模板系统的基础模板，其他页面继承基础模板，添加主体部分就行了。

    至此，一个项目的框架就大致搭建好了，剩下的就是开发了。

  4. 开发
    前后端交互一般使用ajax获取json数据，后台会提供url来获取相关数据，前端使用js或jquery的ajax异步获取数据，然后初始化前端的插件，更新数据等。

现代web开发

这部分内容与实验室无关，仅作为前端开发的补充，有兴趣可以看一下。

因为nodejs的出现，现在的web开发涌现了一大批的自动化开发工具。

  • bower：前端包的管理器，管理jquery，bootstrap等前端框架
  • grunt/gulp：用于重复操作的脚本化，你可以使用其插件压缩源代码，生成生产版本，添加浏览器前缀，监视文件，动态调试等
  • yeoman：一个脚手架工具，使用其生成器可以很快的生成一个前端项目框架，安装grunt，bower，自动下载依赖包等。
  • angularJs：google开发的前端库，可以动态绑定数据显示等。
  • react：Facebook开发的前端组件库，可以定义组件方便重用，而且在大数据量的情况下速度很快。

Js

  • Js的比较：基本不可变的类型进行的是值比较，而对象等可变类型进行的是引用比较，即使内容完全相同，地址不同也不相等（==和===都不相等）
  • JS函数值传递：共享传递，我理解为传递一个引用，不可变类型修改的时候，直接修改引用，但不会影响到外面；可变类型的引用是一个地址，当使用引用修改其属性或方法时，修改了对应的内存，会影响到外部数据，但若使该引用指向别的复合类型（即修改了引用），不会影响到外部【http://www.jb51.net/article/60568.htm】
  • 继承:使用protoType实现继承，详见<>
  • This指针：js里的this指针大多指向调用者，当函数执行再判断this是什么
  • 获取元素对象的方法：getElementById（Class，TagName）
  • Js改变this指针的方法：Bind（），call（），apply（）等（js高级程序设计）
  • 原生ajax的实现-xmlHttpRequest，注意浏览器兼容性，熟练书写
  • Js中每个Element的属性要对的上号，比如offsetTop，offsetHeight，scrollTop，clientHeight巨多
  • JS事件的全部属性
