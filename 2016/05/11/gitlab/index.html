gitLab笔记

gitLab是一个github的ruby实现，有打包好的安装包可以使用，也可以自己配置源码安装，但是很麻烦。我使用安装包安装失败，所以使用了源码安装。

安装

源码安装

  • Ubuntu 源码安装

    • 安装依赖包

          sudo apt-get install -y build-essential zlib1g-dev libyaml-dev libssl-dev libgdbm-dev libreadline-dev libncurses5-dev libffi-dev curl openssh-server checkinstall libxml2-dev libxslt-dev libcurl4-openssl-dev libicu-dev logrotate python-docutils pkg-config cmake nodejs
如果需要使用Kerberos协议来进行用户认证，则安装`libkrb5-dev`

    sudo apt-get install libkrb5-dev

    • git
      需要git 1.7.10以上，使用git --version查看

      sudo apt-get install -y git-core

    • 安装邮件服务
      可以使用其他邮件服务器

      # 需要配置时选择`Internet site`
sudo apt-get install -y postfix

    • ruby

      1. 需要ruby2.1以上，使用源码安装ruby：

        cd ruby-2.1.8
        ./configure --disable-install-rdoc
        make
        sudo make install
2. 安装Bundler Gem

        sudo gem install bundler --no-ri --no-rdoc

    • go
      使用源码安装go：

      sudo tar -C /usr/local -xzf go1.5.3.linux-amd64.tar.gz
sudo ln -sf /usr/local/go/bin/{go,godoc,gofmt} /usr/local/bin/

    • 添加git用户

      sudo adduser --disabled-login --gecos 'GitLab' git

    • 数据库
      可以使用PostgreSQL和MySQL数据库

      * MySQL  
需要MySQL 5.5.14以上，使用`mysql --version`检查  
安装MySQL并进行配置

        # 安装MySql
        sudo apt-get install -y mysql-server mysql-client libmysqlclient-dev
        # 进行安全设置，设置root密码等，gui用户无需
        sudo mysql_secure_installation
        # 登陆MySQL，回车后，输入密码
        mysql -u root -p
        # 创建MySQL用户并设置密码，$password为git密码
        mysql> CREATE USER 'git'@'localhost' IDENTIFIED BY '$password';
        # 设置引擎
        mysql> SET storage_engine=INNODB;
        # 创建GitLab使用的数据库表
        mysql> CREATE DATABASE IF NOT EXISTS `gitlabhq_production` DEFAULT CHARACTER SET `utf8` COLLATE `utf8_unicode_ci`;
        # 为git用户赋予权限
        mysql> GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, CREATE TEMPORARY TABLES, DROP, INDEX, ALTER, LOCK TABLES ON `gitlabhq_production`.* TO 'git'@'localhost';
        # 退出MySQL
        mysql> \q

        # 检测配置是否成功，能登陆成功即可
        sudo -u git -H mysql -u git -p -D gitlabhq_production

安装add_limits_mysql，mysql的列被限制的太小，该软件包可以更改列的类型来缓解

    sudo bundle exec rake add_limits_mysql

  • Redis
    需要Redis 2.8以上

    # 安装Redis
sudo apt-get install redis-server

# 配置Redis使用sockets
sudo cp /etc/redis/redis.conf /etc/redis/redis.conf.orig
# 将port设置为0，使redis不再监听TCP
sed 's/^port .*/port 0/' /etc/redis/redis.conf.orig | sudo tee /etc/redis/redis.conf
# 启用Redis
echo 'unixsocket /var/run/redis/redis.sock' | sudo tee -a /etc/redis/redis.conf
# 赋予redis用户访问sockets的权限
echo 'unixsocketperm 770' | sudo tee -a /etc/redis/redis.conf
# 创建sockets存放目录
mkdir /var/run/redis
chown redis:redis /var/run/redis
chmod 755 /var/run/redis
# 没看懂
if [ -d /etc/tmpfiles.d ]; then
  echo 'd  /var/run/redis  0755  redis  redis  10d  -' | sudo tee -a /etc/tmpfiles.d/redis.conf
fi
# 重启redis服务使配置生效
sudo service redis-server restart
# 把git用户主属组设置为redis
sudo usermod -aG redis git

  • GitLab
    下载GitLab源代码稳定版，解压到安装目录，进行配置

    # 到gitlab目录
cd /home/git/gitlab
# 以git用户身份运行以下命令
# 拷贝gitlab主配置文件
sudo -u git -H cp config/gitlab.yml.example config/gitlab.yml
# 配置主文件
# 1. 设置gitlab -> host为你的域名或ip
# 2. 设置gitlab -> email_from为你的邮箱
# 3. 查看git安装位置（which git），检查git -> bin_path设置是否一致

# 拷贝生成安全配置文件
sudo -u git -H cp config/secrets.yml.example config/secrets.yml
# 其属主应该为git，出于安全考虑，只允许git操作该用户
sudo -u git -H chmod 0600 config/secrets.yml

# 赋予git用户临时文件的访问权限
sudo chown -R git log/
sudo chown -R git tmp/
sudo chmod -R u+rwX,go-w log/
sudo chmod -R u+rwX tmp/
sudo chmod -R u+rwX tmp/pids/
sudo chmod -R u+rwX tmp/sockets/

# 创建uploads文件夹，gitlab-workhorse会使用该文件夹
sudo -u git -H mkdir public/uploads/
# 出于安全考虑仅允许git操作该文件夹
sudo chmod 0700 public/uploads

# 赋予git必要操作权限
sudo chmod -R u+rwX builds/
sudo chmod -R u+rwX shared/artifacts/

# 拷贝生成unicorn配置文件
sudo -u git -H cp config/unicorn.rb.example config/unicorn.rb
# 获取主机核心数
nproc
# 配置worker_processes大于等于核心数，如果内存超过2G，可以设为3

# 拷贝生成rack_attack配置文件
sudo -u git -H cp config/initializers/rack_attack.rb.example config/initializers/rack_attack.rb

# 设置git-autocrlf来处理不同平台的换行符
sudo -u git -H git config --global core.autocrlf input

# 拷贝生成Redis配置文件，必要的话修改redis配置
sudo -u git -H cp config/resque.yml.example config/resque.yml

# 拷贝生成数据库配置文件，这里拷贝mysql的
# sudo -u git cp config/database.yml.postgresql config/database.yml

sudo -u git cp config/database.yml.mysql config/database.yml
# 配置数据库不同模式下的用户名和密码
# 出于安全考虑，只允许git操作配置文件
sudo -u git -H chmod o-rwx config/database.yml

# 安装gitlab必须包，mysql
#该命令需要gitlab根目录的Gemfile等文件，必须在根目录执行，否则会找不到Gemfile错误
#sudo -u git -H bundle install --deployment --without development test mysql aws kerberos
sudo -u git -H bundle install --deployment --without development test postgres aws kerberos

  • 其他配置

    cd /home/git/gitlab
# 初始化数据库
sudo -u git -H bundle exec rake gitlab:setup RAILS_ENV=production
# 添加启动脚本
sudo cp lib/support/init.d/gitlab /etc/init.d/gitlab
# 添加到自启队列
sudo update-rc.d gitlab defaults 21
# 有必要的话配置全局文件，设置gitlab用户，gitlab目录等
sudo cp lib/support/init.d/gitlab.default.example /etc/default/gitlab
# 设置开启日志
sudo cp lib/support/logrotate/gitlab /etc/logrotate.d/gitlab
# 生成网站静态文件
sudo -u git -H bundle exec rake assets:precompile RAILS_ENV=production

# 检查应用状态
sudo -u git -H bundle exec rake gitlab:env:info RAILS_ENV=production
# 重启gitlab
# sudo service gitlab start
# or
sudo /etc/init.d/gitlab restart

  • Web服务器
    可以使用其他服务器，这里使用nginx 

    # 安装nginx
sudo apt-get install -y nginx
# 拷贝gitlab配置文件到nginx目录
sudo cp lib/support/nginx/gitlab /etc/nginx/sites-available/gitlab
# 配置gitlab的nginx配置文件
# 1. 修改server_name为你的IP或域名
# 2. 如果不想作为默认服务器，则删掉
#    listen 0.0.0.0:80 default_server;
#    listen [::]:80 default_server;
#    后的default_server，否则，删除nginx默认服务配置
#    sudo rm -f /etc/nginx/sites-enabled/default

# 链接到实装目录，使得配置生效
sudo ln -s /etc/nginx/sites-available/gitlab /etc/nginx/sites-enabled/gitlab

# 检查配置文件是否正确
sudo nginx -t
# 重启nginx服务
sudo service nginx restart

错误汇总

  • [ruby]安装bundler时报错，缺少方法

    $ sudo gem install bundler
    ERROR: Loading command: install (LoadError) 

    cannot load such file -- zlib

    ERROR: While executing gem … (NoMethodError) 

    undefined method `invoke_with_build_args' for nil:NilClass

    原因：缺少zlib组件或ruby支持
    解决办法：安装组件和ruby支持

    # 安装系统依赖
sudo apt-get install zlib1g-dev
# 安装ruby支持
cd ruby/ext/zlib
ruby ./extconf.rb
make
make install

  • [ruby]Gem在线安装组件时，找不到openssl错误

    Could not load OpenSSL.
    You must recompile Ruby with OpenSSL support or change the sources in your
    Gemfile from ‘https’ to ‘http’.
    Instructions for compiling with OpenSSL using RVM are available at rvm.io/packages/openssl.

    原因：这是安装ruby时，未能成功安装openssl支持造成的，这个是因为make文件中thread_native.h路径定义依赖于top_srcdir变量，而top_srcdir并未在make文件中定义，导致未能正确安装openssl的支持，手动添加即可。
    解决办法：

    1. 确保openssl已安装

      which openssl
# or
sudo apt-get install openssl

    2. 查找thread_native.h文件路径

      $ sudo find / -name thread_native.h 
/home/lql/Desktop/ruby-2.1.5/thread_native.h

    3. 生成并修改openssl三方支持的make文件

      cd ruby/ext/openssl
# 生成make文件
ruby extconf.rb --with-openssl-lib=/usr/local/openssl-1.0.1/lib/ --with-openssl-include=/usr/local/openssl-1.0.1/include/
# 在make文件顶部添加`top_srcdir = ../..`
sudo vi ./ext/openssl/Makefile

    4. 编译安装openssl支持

      # 安装支持
make && make install

  • [ruby]Gem安装时速度太慢，经常报网络错误
    解决办法：换用淘宝源

    # 删除原来的国外源
gem sources -r https://rubygems.org
# 添加淘宝源
# 注意是htpps协议，淘宝源不支持http协议
gem sources -a http://ruby.taobao.org/
# 检查是否只留一个淘宝源
gem source -l
  • [ruby]Gem已修改全局source，但是安装时始终在https://rubygems.org下载镜像
    原因：GemFile中指定了source配置，优先使用该配置。
    解决办法：编辑Gemfile文件，修改source配置为淘宝源http://ruby.taobao.org/即可。

  • [ruby]Gem安装时MySQL相关支持安装出错

    libmysqlclient is missing. You may need to ‘apt-get install libmysqlclient-dev’ or ‘yum install mysql-devel’, and try again.

    Could not create Makefile due to some reason, probably lack of necessary libraries and/or headers.
    Check the mkmf.log file for more details.
    You may need configuration options.

    An error occurred while installing mysql2 (0.3.20), and Bundler cannot continue.
    Make sure that gem install mysql2 -v '0.3.20' succeeds before bundling.

    原因：本机缺少libmysqlclient库
    解决办法：按照提示安装libmysqlclient库

    sudo apt-get install libmysqlclient-dev

  • [nginx]配置文件检查未通过，upstream "gitlab-workhorse"重定义

    $ sudo nginx -t
    nginx: [emerg] duplicate upstream “gitlab-workhorse” in /etc/nginx/sites-enabled/gitlab~:19
    nginx: configuration file /etc/nginx/nginx.conf test failed

    原因：因为报错时显示gitlab~文件，不是正常的gitlab文件
    查看/etc/nginx/sites-enabled下配置文件

    $ ls /etc/nginx/sites-enabled
gitlab  gitlab~

    查看nginx.conf主配置文件 

    include /etc/nginx/conf.d/*.conf;
include /etc/nginx/sites-enabled/*;

    发现其将sites-enabled下所有文件都包含了进去，当然包括gitlab~这个临时文件，相当于gitlab文件被加载了两次，导致重复定义。
    解决办法：删除gitlab~临时文件

    $ sudo rm /etc/nginx/sites-enabled/gitlab~
$ sudo nginx -t
nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
nginx: configuration file /etc/nginx/nginx.conf test is successful
  • [nginx]配置文件检查未通过，default server重定义
    原因：nginx安装后会生成一个默认配置文件在/etc/nginx/sites-enabled/default,该文件定义了一个default server，用户再次定义就导致了重复定义。
    解决办法：删除/etc/nginx/sites-enabled/default配置文件，或删除其中的default server配置项。
  • gielab配置完成后，访问网站127.0.0.1始终得到welcome nginx页面
    原因：浏览器缓存导致。
    解决办法：清空缓存。

  • gitlab检查时无法连接gitlab-shell,检查失败

    Check GitLab API access: FAILED: Failed to connect to internal API gitlab-shell self-check failed
    Try fixing it:
    Make sure GitLab is running;
    Check the gitlab-shell configuration file:
    sudo -u git -H editor /home/git/gitlab-shell/config.yml
    Please fix the error above and rerun the checks.

    原因：gitlab连接gitlab-shell和gitlab-workhorse需要用到三个配置文件 

    /home/git/gitlab/config/unicorn.rb
    listen "127.0.0.1:8080", :tcp_nopush => true
/home/git/gitlab/config/gitlab.yml
  gitlab:
    host: localhost
    port: 80
/home/git/gitlab-shell/config.yml
    gitlab_url: http://localhost:8000/

    这三个配置文件里的host:port不一致（默认不一致）导致无法联动。
    解决办法：修改配置文件使其一致即可

  • 访问网站的502错误
    原因：在虚拟中测试只分配了1G内存，而在配置文件中设置的线程数为3，导致内存不足。
    解决办法：修改gitlab/config/unicorn.rb配置文件，将worker_processes设置为1，重启服务sudo service gitlab restart。
