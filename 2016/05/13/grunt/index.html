Grunt笔记

grunt是基于node的js构建工具。对于需要反复重复的任务，例如压缩（minification）、编译、单元测试、linting等，自动化工具可以减轻你的劳动，简化你的工作。

安装

  • nodeJs

    # 安装 CLI
npm install grunt-cli

    注意，安装grunt-cli并不等于安装了Grunt！Grunt CLI的任务很简单：调用与Gruntfile在同一目录中 Grunt。这样带来的好处是，允许你在同一个系统上同时安装多个版本的 Grunt，每个项目使用自己目录下的Grunt。

    在你的项目文件夹下安装grunt，并添加到依赖列表。

    npm install grunt --save

Gruntfile.js

Gruntfile.js是grunt的配置文件，该文件定义了多个任务并对任务进行配置，使其满足不同的需求；grunt运行时会默认查找该文件。
一个简单的文件示例：

'use strict';    // 使用严格模式，可选
// 遵循cmd规范，声明为一个包
module.exports = function(grunt){

    // 配置对象，自定义，放在最前面方便修改和查看
    var config = {
        app:'app',
        dist:'dist',
    };

    // 初始化配置，每项名称可以被看做为任务名
    grunt.initConfig({
        // 传入先前定义的配置变量
        config:config,
        // 使用内置函数读取json文件为属性
        pkg: grunt.file.readJSON('package.json'),

        // copy任务的定义与配置
        copy:{
            // 插件配置，可选，用来控制插件，覆盖其配置的默认值
            options: {},
            // copy任务的html对象，可以认为是对html的拷贝
            html:{
                // copy插件配置，可选，优先级高于任务级
                options: {},
                // 指明源文件，这里使用内置模板调用了config.app的值
                src:'<%= config.app %>/index.html',
                // 指明，目的文件
                dest:'<%= config.dist %>/index.html',
            }
        },
        // clean任务的定义与配置
        clean:{
            html:{
                // 因为是删除操作，只需要源文件即可
                src:'<%= config.dist %>/index.html',
            }
        },
    });
    // 加载插件，加载完成就可以用grunt copy执行任务了
    grunt.loadNpmTasks('grunt-contrib-copy');
    grunt.loadNpmTasks('grunt-contrib-clean');

    // 注册任务，这里一般用来将多个任务结合起来，当执行build任务时，clean，copy会先后被调用
    grunt.registerTask('build', ['clean','copy']);
};

运行任务

运行任务时，请确保grunt和其使用的插件全部被安装。

  1. 执行默认任务
    当注册一个名为default的任务后，在项目文件夹执行grunt，default任务就会被执行。

    // 定义default任务
grunt.registerTask('default', ['jshint', 'qunit', 'concat', 'uglify']);

# 执行grunt任务
$ grunt

  2. 指定任务名
    这里的任务名可以是初始化配置时的属性，也可以是注册的任务别名。

    // 属性名
grunt.initConfig({
    // clean任务
    clean:{
        html:{
            src:'index.html',
        }
    },
})

// 注册的任务别名
grunt.registerTask('build', ['clean', 'build']);

# 执行任务
$ grunt clean
$ grunt build

  3. 指定目标
    目标是相对于任务的，即任务的目标。grunt默认会将任务中所有的目标任务都执行，指定目标后仅执行指定目标的任务。

    // 定义具有html和css两个目标的copy任务
grunt.initConfig({
    copy:{
        html:{
            src:'app/index.html',
            dest:'app/index.html',
        },
        css:{
            src:'app/main.css',
            dest:'app/main.css',
        },
    },
})

# 仅拷贝html
$ grunt copy:html

# 仅拷贝css
$ grunt copy:css

文件配置

大多的任务都是执行文件操作，Grunt有一个强大的抽象层用于声明任务应该操作哪些文件。这里有好几种定义src-dest(源文件-目标文件)文件映射的方式，均提供了不同程度的描述和控制操作方式。

声明源文件和目标文件

  1. 简洁格式
    使用src和dest指明源文件和目的文件，可以为字符串和数组。一般用于只需要指定单个属性（src或dest）的任务。

    grunt.initConfig({
  concat: {
    bar: {
      src: ['src/bb.js', 'src/bbb.js'],
      dest: 'dest/b.js',
    },
  },
});

  2. 文件对象格式
    使用files属性的键值来指明源文件和目的文件，键只能为字符串，值可以为字符串或数组，一般用来指明一对多的关系。

    grunt.initConfig({
  concat: {
    foo: {
      files: {
        'dest/a.js': ['src/aa.js', 'src/aaa.js'],
        'dest/a1.js': ['src/aa1.js', 'src/aaa1.js'],
      },
    }
  },
});

  3. 文件数组格式
    files属性为多个字典的数组，可以用来指明多对多的关系。

    grunt.initConfig({
  concat: {
    foo: {
      files: [
        {src: ['src/aa.js', 'src/aaa.js'], dest: 'dest/a.js'},
        {src: ['src/aa1.js', 'src/aaa1.js'], dest: 'dest/a1.js'},
      ],
    },
  },
});

通配符

文件的指定可以使用通配符来进行匹配，支持的通配符：

    • 匹配任意数量的字符，但不匹配 /
  • ? 匹配单个字符，但不匹配 /
  • ** 匹配任意数量的字符，包括 /，只要它是路径中唯一的一部分
  • {} 允许使用一个逗号分割的“或”表达式列表
  • ! 在模式的开头用于排除一个匹配模式所匹配的任何文件

插件

插件名 描述
load-grunt-tasks 用于任务的加载
grunt-contrib-copy 拷贝文件
grunt-contrib-clean 文件删除
grunt-contrib-uglify 压缩JS文件
grunt-contrib-qunit 测试html文件
grunt-contrib-concat 合并多个js文件
grunt-contrib-jshint 检测js文件语法和代码风格
grunt-contrib-watch 监视指定的文件，改动时执行指定任务
grunt-contrib-sass 将scss/sass文件编译为css文件
