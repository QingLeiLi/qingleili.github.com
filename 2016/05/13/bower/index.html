bower笔记

Bower是一个包管理程序，可以很方便的安装需要的文件。但是bower一般结合grunt一起使用，解决bower引用文件麻烦的局限性。

安装

npm install -g bower

##　使用

  1. 初始化项目
    在项目目录下运行bower init命令，bower会通过交互新建一个配置文件-bower.json，该文件设置了项目的一些信息以及依赖的软件包。

    bower init

    通过交互设置大致信息后，生成配置文件

  2. 安装当前依赖的软件包
    使用bower install来安装当前项目所需的依赖，bower会读取dependencies配置项，安装所有依赖。

    bower install

  3. 安装开发环境软件包
    使用bower install angular --save-dev命令安装angular，参数--save-dev表示将该软件表作为开发依赖放入devDependencies配置项

    bower install angular --save-dev

  4. 安装生成环境软件包
    使用bower install angular --save命令安装angular，参数--save表示将该软件表作为生产依赖放入dependencies配置项

    bower install angular --save

  5. 使用安装完的组件
    bower软件包的使用可以是很笨的方法，直接引用文件，很麻烦，一般结合grunt使用。

    <!-- jquery -->
<script src="../bower_components/jquery.min/index.js"></script>

    安装软件包

    Bower提供了多种软件包的安装方式，可以通过注册名，github短地址，url安装。Bower安装的软件包默认在./bower_components/目录。

  • 注册名
    bower会在其注册软件列表中查找该软件包的名字，查找到后会判断是否缓存，然后去指定位置获取软件包并安装。

    bower install jquery

  • github短地址
    若所需的软件包没有在bower列表中注册，且该软件包位于github仓库中，则可以使用该方式安装。使用时提供一个用户名和仓库名的组合，比如jquery/jquery，就表示该软件包位于jquery用户的jquery仓库中，这样就定位了唯一一个仓库，bower会去拉取源代码安装。

    bower install jquery/jquery

  • url安装
    如果软件包既没有进行注册，也可以使用url直接进行安装，例如jquery的cdnhttp://cdn.bootcss.com/jquery/3.0.0-beta1/jquery.min.js,bower下载该文件将其放入jquery.min目录或者github的完整地址。

    bower install http://cdn.bootcss.com/jquery/3.0.0-beta1/jquery.min.js
bower install https://github.com/jquery/jquery

配置文件

bower的配置文件大致有两个：bower.json和.bowerrc。

  • bower.json
    该文件置于项目根目录，配置的是项目相关的一些基本信息以及依赖的软件包。

    # bower.json
{
  "name": "test",
  "authors": [
    "lql <110@qq.com>"
  ],
  "description": "a test project fo bower",
  // 项目依赖的入口文件
  "main": "",
  "keywords": [
    "test"
  ],
  "license": "MIT",
  "homepage": "",
  // 表示私有，不会注册到bower的列表中
  "private": true,
  // 忽略的文件及文件夹
  "ignore": [
    "**/.*",
    "node_modules",
    "bower_components",
    "test",
    "tests"
  ],
  // 依赖，可以使用`bower install`直接安装这些包
  "dependencies": {
    "bootstrap": "^3.3.6",
    "jquery.min": "http://cdn.bootcss.com/jquery/3.0.0-beta1/jquery.min.js"
  },
  // 开发环境使用的依赖
  "devDependencies": {
    "bootstrap": "^3.3.6",
    "jquery.min": "http://cdn.bootcss.com/jquery/3.0.0-beta1/jquery.min.js",
    "angular":"~1.3.2",
  }
}

  • .bowerrc
    该文件也是置于项目根目录，设置的是该项目下bower的个性化配置。

    #.bowerrc 
{
    // 软件包的安装文件夹，默认是bower_components
    "directory":"bower_components",
    // http代理地址，bower会通过该代理拉取软件包
    "proxy":"http://proxy.example.com:8080",
    // https的代理地址
    "https-proxy":"https://proxy.example.com:8080",
    // 超时时间
    "timeout":60000,
}

错误汇总

  • 初始化错误

    $ bower init
    bower ENOINT Register requires an interactive shell

    Additional error details:
    Note that you can manually force an interactive shell with –config.interactive

    原因：使用gitbash或cygwin运行命令会有问题，具体问题没细查。
    解决办法：

    • 在cmd或标准终端下运行命令
    • 重装gitbash
      重装时注意配置git bash。
      1. Use Git form the Windows Command Prompt
      2. Checkout Windows-style, commit Unix-style line endings
      3. Use Windows’ default console window
