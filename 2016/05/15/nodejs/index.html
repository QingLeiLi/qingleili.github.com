NodeJs笔记

安装nvm（nodejs版本控制）

nodejs有两个版本控制软件，n与nvm。

  • nvm
    独立于nodejs，可以安装多个版本nodejs并可以切换。
    使用nvm安装的nodejs都被安装在nvm自己的目录下，通过修改环境变量来切换不同的版本；而且某个版本下的包都是安装在当前node版本文件夹下的，切换版本后，所有的node包需要重新安装。
    nvm是一个将各个node完全隔离（全局的软件包也是独立的）开的沙盒，尽可能减少了不兼容的情况发生。
    不支持windows，但是其实现了windows平台的nvmw。
  • n
    作为nodejs的一个开源包，需要使用nodejs全局安装，如果要使用n，你首先要有一个nodejs环境。
    其依赖于node，通过修改/usr/local/bin来切换不同版本；各个版本的全局包是共用的，可能产生不兼容的问题。

nvm安装

  1. 获取nvm，可以使用git，curl等获取，这里使用git将源码下载到~/.nvm目录

    git clone https://github.com/creationix/nvm.git ~/.nvm
cd ~/.nvm
# 切换分支到最新
git checkout `git describe --abbrev=0 --tags`

  2. 运行其脚本-nvm.sh，设置nvn运行所需的环境变量等信息

    source ~/.nvm/nvm.sh

  3. 如果用户注销或关机，环境变量会失效，需要重新运行nvm.sh，为避免繁琐，将其写入环境变量文件。
    在~/.bashrc,~/.profile,~/.zshrc选择一个写入即可。这里.即为source，是等效的。

    export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && . "$NVM_DIR/nvm.sh" # This loads nvm

nvm使用

  • nvm ls：查看已安装的node；
  • nvm ls-remote：查看可安装的node版本；
  • nvm install 6.1.0：安装node6.0.1版本，nvm会尝试下载node的二进制文件进行安装，如果失败，则尝试源码安装；
  • nvm uninstall 6.1.0：卸载6.10.版本
  • nvm use [6.1.0]：nvm会检测当前目录下是否有.nvmrc文件，有的话使用指导版本，否则要指明使用的版本号

cnpm

npm是node的包管理器，用于自动解决依赖，但因为npm的软件仓库在国外，导致npm安装包经常失败，为此淘宝建立了npm的镜像仓库，用于国内安装加速。
cnpm就是使用国内镜像用于替换npm命令的，其基于npm，只是npm的封装。

安装cnpm

npm install -g cnpm --registry=https://registry.npm.taobao.org

cnpm使用

cnpm的使用与npm大致一直，有两点不同：

  1. 淘宝不支持发布软件包，所以publish命令不能使用；

  2. cnpm支持额外命令-sync，用于立即同步某个模块，淘宝会连接npm软件包仓库，将最新版本拉取到镜像仓库。

    $ cnpm sync connect

package.json

package.json是nodejs项目的配置文件，定义了整个项目的名称，版本等基本信息以及运行所需依赖等信息。

  • devDependencies和dependencies
    运行项目所需依赖，分别为开发环境所需依赖和生产环境所需依赖，为字典格式，使用npm install时，npm会读取这两个字段，将所有依赖进行安装。版本的声明有多种模式：

    1. 宽松版本限制
      使用^表示，说明所安装软件包版本要符合要求版本的主版本号即可。主版本号为版本号的第一位数字。如下，项目依赖1.0.1版本的grunt，但是使用了^宽松版本限制，可以使用1.0.0~1.9.9所有版本的grunt。

      "devDependencies": {
    "grunt": "^1.0.1"
}

    2. 严格版本限制
      使用~表示，说明所安装软件包版本要符合要求版本的最小版本。最小版本号为版本号的最后一位位数字。如下，项目依赖1.0.1版本的grunt，但是使用了~严格版本限制，只可以使用1.0.0~1.0.9所有版本的grunt。

      "devDependencies": {
    "grunt": "^1.0.1"
}
