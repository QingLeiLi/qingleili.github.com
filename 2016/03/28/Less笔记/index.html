Less笔记

Less是一个CSS预处理语言，在Less中可以使用变量，函数，嵌套等特性，方便维护，Bootstrap源码使用了该工具。

安装

  • 浏览器

    1. 下载Less源码，解压压缩版本less.js/dist/less.min.js.

    2. 在页面中引入Less.js，它会把页面中所有less文件解析为css后压入页面。

      <script src="less.min.js" type="text/javascript"></script>

    3. 引入你的Less文件，注意rel属性设置为stylesheet/less.

      <link rel="stylesheet/less" type="text/css" href="styles.less" />

      注意：

    • 所有Less文件应该在Less.js之前加载，否则less.js不会解析在其之后的less文件。

    • 配置Less.js
      有三种方式可以配置Less.js，全局变量，Script标签变量和link标签变量
      优先级：全局变量 < Script标签变量 < link标签变量

      • 全局变量
        在引入less.js之前引入less全局变量来配置它。

        <script>
  less = {
    env: "development",
  };
</script>
<script src="less.min.js" type="text/javascript"></script>

      • 标签变量
        使用标签引入配置时，需要加data前缀，而且将驼峰命名改为短线，eg:logLevel -> data-log-level。
        非字符串的值必须符合json格式，并且需要浏览器支持JSON.parse()，eg：

        data-global-vars='{ "myvar": "#ddffee", "mystr": "\"quoted\"" }'

        • Script标签变量
          在引入less.js文件的script标签上添加数据属性也可以来配置它。

          <script src="less.min.js" data-env="development" type="text/javascript"></script>

        • link标签变量
          与script标签类似但只支持render time配置项

          <link data-env="development" href="less/styles.less">

    • API
      允许通过js对less进行有限的操作。

      • 监视模式 - watch()
        当启用监视模式后，js会监视less文件，每当less文件变化会自动刷新客户端。
        启用监视器需要less的env配置项为development。
        两种方式启用监视模式。

        • js
          在引入less.js后，执行less.watch()即可启用监视模式。

          <script>less.watch();</script>

        • url
          在网页url中添加hash-#!watch来启用监视模式

          http://127.0.0.1/index.html#!watch

      • 编辑less变量 - modifyVars()
        允许使用js编辑less的内部变量，变量改变后，less会自动编译，无需重载。

        less.modifyVars({
  '@buttonFace': '#5B83AD',
  '@buttonText': '#D9EEF2'
});

    • 选项

      • async [false]
        是否异步加载import的文件，添加async选项。
      • fileAsync [false]
        当使用file://协议时，是否异步获取import的文件。
      • dumpLineNumbers [‘’]
        向生成的css文件中添加生成每条规则语句的行号(less文件中的行)。
        • ‘comments’
          添加行号和注释。
        • ‘mediaquery’
          与火狐插件相关。
        • ‘all’
          以上两项
      • env
        默认值取决于url，当使用file://协议，使用本地地址(127.0.0.1 or localhost)或不是80端口，会自动设置为development；其他则是production。
        • ‘production’：css缓存在本地，终端不会显示less编译的信息
        • ‘development’:显示编译相关信息。
      • errorReporting [‘html’]
        html | console | function
        当编译错误时，错误的显示方法。

      • functions {}
        定义可以在less文件中使用的函数。

        less = {
    functions: {
        // 使用myfunc()函数
        myfunc: function() {
            return new(less.tree.Dimension)(1);
        }
    }
};
      • logLevel [2]
        定义终端的信息输出等级，如果是production模式，不会输出任何信息。
        • 0
          无输出
        • 1
          只输出错误
        • 2
          输出错误和额外信息
      • poll [1000]
        如果是监视模式，定义两次请求比较文件的间隔，但未是ms。
      • relativeUrls [false]
        import的文件是否使用相对路径，false-是，true-否。

      • globalVars [undefined]
        定义相对于less的全局变量，在less文件中可以覆盖该处定义。

        less.globalVars = { myvar: "#ddffee", mystr: "\"quoted\"" };
      • modifyVars [undefined]
        编辑less的内部变量，变量改变后，less会自动编译，无需重载。
      • rootpath [false]
        添加到每个资源头部的地址。
      • useFileCache [true]
        缓存less文件，这样，调用modifyVars等属性时不会再次请求。

    • 函数
      http://www.1024i.com/demo/less/reference.html

  • NodeJs

使用

变量

定义全局变量可以方便后续维护，并且可以使用局部变量来覆盖全局变量。

//变量定义
@baseColor : #4D926E;
//使用变量
a {
    font-color : @baseColor;
}

  • 变量可以连续解析

    a : 13;
b : 'a';
content : @@b; //13

  • 变量的查找是当前作用域最后被赋予的值；若当前作用域没有定义，则搜索父作用域，直至找到该变量。

    @var: 0;
.class1 {
    @var: 1;
    .class {
        @var: 2;
        //最后定义是下一行
        three: @var;
        @var: 3;
    }
    one: @var;
}

  • 变量替换的解析是普通变量解析的重复；先解析第一个变量，再从当前作用域开始解析被替换的变量

    .lazy-eval-scope {
    // %9
    width: @var;
    @a: 9%;
}

@var: @a;
@a: 100%;

混合（Mixins）

Less允许直接嵌入类或函数来实现代码复用，因为函数内容较多另起一段。

//定义一个类
.bordered {
    border-top: dotted 1px black;
    border-bottom: solid 2px black;
}
#test {
    color: #111;
    //直接套用类，bordered的所有属性会被附加到里面
    .bordered;
}

  • 复用嵌套内部的类

    #test {
    color: #111;
    // 定义内部类
    .bordered {
        border-top: dotted 1px black;
        border-bottom: solid 2px black;
    }
}
.test {
    // 复用嵌套内部的类
    #test > .bordered;
}

  • 关于important
    在混合后面添加!important参数表示该混合中的所有属性都添加important

    .mixin {
    // 混合中所有的属性都加important
    .rounded-corners !important;
}

嵌套

Less允许在某一项下面嵌套来实现子类选择器。

#test {
    // 编译为 #test a{}
    a {
        // 编译为#test a:hover
        &:hover {
            border-width: 1px;
        }
    }
}

  • 关于&
    我理解的&代表父级的所有元素，当父级元素是用逗号隔开的多个元素时，&则是其迭代。

    a {
    // 编译为#test a:hover
    &:hover {}
    // 编译为 .parent #test
    .parent&{}
}
// 多个则迭代
#test , .class {
    //编译为
    // #test + #test {}
    // #test + .class {}
    // .class + .class {}
    // .class + #test  {}
    & + & {}
}

函数

带有参数的混合。

  • 函数名可以是类或者ID，但不可以是纯字符串的函数名。

    // 正确
#test(@radius: 5px) {
    border-radius: @radius;
}
// 正确
.test(@radius: 5px) {
    border-radius: @radius;
}
// 错误
test(@radius: 5px) {
    border-radius: @radius;
}

  • 带有参数的函数并不会作为样式应用于相关类上

    // 其带有参数，test类不会具有圆角
.test(@radius: 5px) {
    border-radius: @radius;
}
// 该行代码才使其具有了圆角
.test(@radius: 5px) {
    .test;
}

  • 使用混合时，所有符合参数要求的函数都会被使用

    // 定义一参无默认值的类
.test(){
    font-size : 11px;
}
// 定义二参有默认值的类
.test(@a,@b : 20px){
    color : @a;
    font-size : @b;
}
#test {
    //上面的两个类都会被应用到此处
    .test(20);
}

  • 调用类时参数个数的计算
    参数的分隔符可以是’,’或’;’,且分号的优先级要高于逗号，当传参里有分号，则分隔符为分号，没有则为逗号。

    // 三个参数，逗号此处为一组值的分割符而不是参数分隔符
.name(1, 2, 3; a, b)；
// 三个参数
.name(1, 2, 3)；

  • 参数可以有默认值

    //设置默认值
.rounded-corners (@radius: 5px) {
    border-radius: @radius;
}
#test {
//使用默认值的类
.rounded-corners；
//覆盖默认值
.rounded-corners (10px)
}

  • 关于@arguments和@rest
    @arguments包含了传入的所有变量，可以用来快速设置有多个值的属性。

    .box-shadow (@x, @y, @blur, @color) {
    box-shadow: @arguments;
}

    如果在一个变量后面添加’…’，即代表该参数代表在此往后的所有参数

    .mixin (@a, @rest...) {
    // @rest 表示 @a 之后的参数
    // @arguments 表示所有参数
}

  • 函数写死第一个参数来控制函数行为
    若函数定义时第一个参数并不是变量，而是值，则调用时该参数与值相等才可以匹配该函数。

    // 当第一个参数是dark时，这条规则才会被应用
.mixin (dark, @color) {
    color: darken(@color, 10%);
}

Guards表达式

类似if/else语句，仅仅是为了符合CSS风格，设计成when。
支持的比较符：大于(>)，大于等于(>=)，等于(=)，小于等于(<=)，小于(<)。

// 当参数大于5回应用该规则
.mixin (@a) when (@a >= 5) {
    background-color: black;
}

  • 并列条件，逗号
    Less并没有or关键字，使用’，’来分割条件，只要有一个为true，则应用规则。

    // 参数大于10或小于-10
.mixin (@a) when (@a > 10), (@a < -10) {}

  • and 和 not
    Less里有and 和not关键字，来实现’和’与’非’。

    // 参数大于10且小于20
.mixin (@a) when (@a > 10) and (@a < 20) {}
// 参数不小于10
.mixin (@a) when not(@a > 10){}

  • 关于’真’和’true’
    Less里面唯一一个是真的值，其他都为假。

    // 一下两个等价
.truth (@a) when (@a) {}
.truth (@a) when (@a = true) {}

    引入

    Less允许导入其他less文件和css文件，只要不是以css结尾的文件都会被认为是less文件。
    Less会首先将import提到当前作用域顶部，然后将import的less文件原样添加到该处，一起编译；并不会去改变css。

    @import “lib.css”;

  • 带有媒体查询的import
    该文件下所有规则都会在该媒体查询内部。

    @import "library.less" screen and (max-width: 400px);

  • 强制指定处理方式 - css | less

    @import (css) "lib";
@import (less) "lib.css";

  • 字符串连接变量
    使用{}括住变量，以防混淆。

    background-image: url("@{base-url}/images/bg.png");

  • 关于~-避免编译
    在css语句前添加~可以 使less不编译该语句。

    .class {
    filter: ~"ms:alwaysHasItsOwnSyntax.For.Stuff()";
}

  • 选择器插值-使用变量指定选择器

    @name: blocked;
.@{name} {
    color: black;
}

  • Less可以使用js

    @height: `document.body.clientHeight`;
