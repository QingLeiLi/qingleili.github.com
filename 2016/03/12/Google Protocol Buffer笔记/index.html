Google Protocol Buffer笔记

Protocol Buffer(以下简称PB)是google的一种数据交换的格式，它独立于语言，独立于平台，采用二进制进行数据存储，相较于xml更高效，但可读性差，原生API有Python，Java和C++，但正在不断被拓展。

安装

依赖

  • 无

安装

我的Ubuntu 12.04自带了peotoc2

  1. 在github下载Google Protocol Buffer源码，下载地址
  2. 根据radme编译安装protoc
    • 运行./autogen.sh脚本生成config文件,脚本运行可能会缺少必要工具，按需安装
    • 编译安装./configure && make && make check && make install
    • 运行protoc --version查看是否安装成功

  3. 安装python支持
    protoc需要Python 2.6或以上，使用python -V检查
    安装python支持前请确保protoc以正确安装，且版本跟python支持版本一致，使用protoc --version检查

    • 进入源代码/python目录

      cd python

    • 进行编译测试

      python setup.py build
#测试可能不通过，酌情是否安装
python setup.py test

    • 安装

      python setup.py install

      或使用pip安装

      sudo pip install -U protobuf==3.0.0a3

使用

protoc的使用

  1. 首先需要编写.proto文件，定义消息的数据结构

    #addressbook.proto
#包名，区分不同的工程，在python中没有作用
package tutorial;
#定义一个数据结构Person
message Person {
  required string name = 1;
  required int32 id = 2;
  optional string email = 3;

  enum PhoneType {
    MOBILE = 0;
    HOME = 1;
    WORK = 2;
  }

  message PhoneNumber {
    required string number = 1;
    optional PhoneType type = 2 [default = HOME];
  }

  repeated PhoneNumber phone = 4;
}
#另一个数据结构AddressBook
message AddressBook {
  repeated Person person = 1;
}

  2. 使用protoc编译.proto文件生成对应语言的类

    #-I参数设置输入目录，--python_out设置输出目录，且输出的是python类，最后一个为相对于输入目录.proto文件的位置
protoc -I=. --python_out=. ./addressbook.proto
#编译为CPP
#protoc -I=. --cpp_out=. msg.proto

    protoc会生成addressbook_pb2.py文件，pb2后缀说明为protoc2版本

  3. 使用类

    #引入类
import addressbook_pb2
#生成一个电话簿对象
address_book = addressbook_pb2.AddressBook()
#为电话簿添加一个person，返回该person
person = address_book.person.add()
#设置person的值
person.name = 'test'
person.id = 1
person.email = 'test@test.com'
#使用字符串反向格式化类，赋值给address_book
address_book.ParseFromString(f.read())
#将address_book格式化为字符串
address_book.SerializeToString()

不同语言序列化与反序列化函数

  • Python

    #序列化
person.SerializeToString()
#反序列化
person.ParseFromString(strData)

  • C++

    //序列化
person.SerializeToOstream()
//反序列化
person.ParseFromIstream(stream)

  • Java

    //序列化
Serialize()
//反序列化
Deserialize()

应用

电话簿的读写

  1. 建立.proto文件

    package tutorial;

message Person {
  required string name = 1;
  required int32 id = 2;
  optional string email = 3;

  enum PhoneType {
    MOBILE = 0;
    HOME = 1;
    WORK = 2;
  }

  message PhoneNumber {
    required string number = 1;
    optional PhoneType type = 2 [default = HOME];
  }

  repeated PhoneNumber phone = 4;
}

message AddressBook {
  repeated Person person = 1;
}

  2. 编译.proto文件

    protoc -I=. --python_out=. ./addressbook.proto

  3. 实现writer，提供一个参数-已存在的电话簿，没有则新建，向电话簿中新加一条记录，eg：python writer.py book

    #writer.py
#! /usr/bin/python        
import addressbook_pb2
import sys

# This function fills in a Person message based on user input.
def PromptForAddress(person):
  person.id = int(raw_input("Enter person ID number: "))
  person.name = raw_input("Enter name: ")

  email = raw_input("Enter email address (blank for none): ")
  if email != "":
    person.email = email

  while True:
    number = raw_input("Enter a phone number (or leave blank to finish): ")
    if number == "":
      break

    phone_number = person.phone.add()
    phone_number.number = number

    type = raw_input("Is this a mobile, home, or work phone? ")
    if type == "mobile":
      phone_number.type = addressbook_pb2.Person.MOBILE
    elif type == "home":
      phone_number.type = addressbook_pb2.Person.HOME
    elif type == "work":
      phone_number.type = addressbook_pb2.Person.WORK
    else:
      print "Unknown phone type; leaving as default value."

if __name__=='__main__':
  # Main procedure:  Reads the entire address book from a file,
  #   adds one person based on user input, then writes it back out to the same
  #   file.
  if len(sys.argv) != 2:
    print "Usage:", sys.argv[0], "ADDRESS_BOOK_FILE"
    sys.exit(-1)

  address_book = addressbook_pb2.AddressBook()

  # Read the existing address book.
  try:
    f = open(sys.argv[1], "rb")
    address_book.ParseFromString(f.read())
    f.close()
  except IOError:
    print sys.argv[1] + ": Could not open file.  Creating a new one."

  # Add an address.
  PromptForAddress(address_book.person.add())

  # Write the new address book back to disk.
  f = open(sys.argv[1], "wb")
  f.write(address_book.SerializeToString())
  f.close()

  4. 实现reader，提供一个参数-已存在的电话簿，输出已存在的记录，eg：python reader.py book

    #reader.py
#! /usr/bin/python

import addressbook_pb2
import sys

# Iterates though all people in the AddressBook and prints info about them.
def ListPeople(address_book):
  for person in address_book.person:
    print "Person ID:", person.id
    print "  Name:", person.name
    if person.HasField('email'):
      print "  E-mail address:", person.email

    for phone_number in person.phone:
      if phone_number.type == addressbook_pb2.Person.MOBILE:
        print "  Mobile phone #: ",
      elif phone_number.type == addressbook_pb2.Person.HOME:
        print "  Home phone #: ",
      elif phone_number.type == addressbook_pb2.Person.WORK:
        print "  Work phone #: ",
      print phone_number.number
if __name__=='__main__':
  # Main procedure:  Reads the entire address book from a file and prints all
  #   the information inside.
  if len(sys.argv) != 2:
    print "Usage:", sys.argv[0], "ADDRESS_BOOK_FILE"
    sys.exit(-1)

  address_book = addressbook_pb2.AddressBook()

  # Read the existing address book.
  f = open(sys.argv[1], "rb")
  address_book.ParseFromString(f.read())
  f.close()

  ListPeople(address_book)

错误汇总

  • 找不到google mock，自动下载失败（GWF）

    Google Mock not present. Fetching gmock-1.7.0 from the web…

    解决办法：手动下载gmock-1.7.0.zip文件，放到源文件根目录，编辑autogen.sh脚本，将自动下载命令注释掉，后续脚本会自动解压安装

    #curl $curlopts -O https://googlemock.googlecode.com/files/gmock-1.7.0.zip

  • 缺少autoreconf相关工具

    ./autogen.sh: 4: autoreconf: not found`

    解决办法：手动安装

    sudo apt-get install autoconf automake libtool

  • autoreconf失败

    autoreconf:automake failed with exit status:1

    解决办法：未解决，但是我又重新运行了一遍就好了…

  • 找不到动态链接库

    protoc: error while loading shared libraries: libprotobuf.so.7: cannot open shared object file: No such file or directory

    解决办法：编辑/etc/ld.so.conf配置文件，添加protoc的链接库目录（默认/usr/local/lib）

    include /etc/ld.so.conf.d/*.conf /usr/local/lib

    执行ldconfig更新库文件路径

    sudo ldconfig

    另一种解决方案，使用环境变量： 

    修改`~/.profile  `配置文件，添加

#Linux环境变量名，该环境变量主要用于指定查找共享库（动态链接库）时除了默认路径之外的其他路径(该路径在默认路径之前查找) 。
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/lib

    使更改生效

    source ~/.profile
