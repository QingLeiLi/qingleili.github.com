Git笔记

安装

依赖

  • 无

安装

  • Ubuntu

    sudo apt-get install git

使用

基本命令

  • git init
    将当前目录初始化为仓库
  • git clone url
    clone一个git仓库到当前目录
  • git add readme.txt(.)
    将readme(所有文件)加入暂存区，新添和修改时都需要add
  • git status
    查看当前文件的改动-与上次commit比较
  • git commit -m “提交注释”
    将改动提交到版本库
  • git rm readme.txt
    在版本控制中移除文件

配置

配置有三个优先级：local>global>system 

local：仅对该仓库  
global：对该用户的所有仓库  
system：对所有仓库
  • git config –global user.name［ qingleili］
    查看［设置］用户名
  • git config –local user.email[ test@qq.om]
    查看［设置］邮箱
  • git config –global color.ui auto
    git颜色显示-自动
  • git config receive.denyCurrentBranch ignore
    允许远程推送

历史与回滚

  • git log
    查看提交历史
  • git reflog
    查看操作历史，可以找回被回滚的版本ID
  • git diff readme.txt
    查看readme文件与版本库里的改动
  • git checkout – readme.txt
    撤销该文件在工作区中的修改到暂存区的状态，若暂存区没有该文件则恢复到版本库中的状态，文件名前有空格
  • git reset HEAD [readme.txt]
    将暂存区中（所有或某个文件）的数据回退到工作区
  • git reset –hard HEAD^
    回滚版本，HEAD表示当前版本，加一个^表示上一个版本，太多的话使用HEAD~100,也可是使用版本Id

远程仓库

  • git pull
    将远程仓库的代码合并到本地
  • git push
    将本地代码提交到远程仓库
  • git push -u origin master
    第一次推送分支要加-u参数，将本地库与远程库进行关联
  • git remote add origin git@github.com:git用户名/learngit.git
    添加远程仓库
  • git remote[ -v]
    查看远程库信息[详细信息]

分支

  • git checkout -b dev
    创建名为dev的分支，并切换到dev分支
  • git branch dev
    创建名为dev的分支
  • git checkout dev
    切换到dev分支
  • git branch
    查看所有分支和当前分支
  • git merge dev
    将dev分支合并到当前分支
  • git branch -d dev
    删除dev分支
  • git checkout -b dev origin/dev
    创建远程分支到本地，clone完后只能看到master分支
  • git branch –set-upstream-to origin/dev dev
    创建远程库与本地库的连接

现场

  • git stash
    保存工作现场
  • git stash list
    查看工作现场列表
  • git stash pop
    恢复工作现场并删除保存的工作现场
  • git stash apply
    恢复工作现场
  • git stash drop
    删除保存的工作现场

其他

  • git获取远程仓库的archive

    git archive master --remote=git@172.26.253.30:/rep/test.git

  • 获取干净代码，不带.git文件夹

    git archive master --format=tar --remote=git@172.26.253.30:/rep/test.git | tar xf -

错误汇总

  • Git Push unpacker 错误

    fatal: failed to write object
    error: unpack failed: unpack-objects abnormal exit To ssh://git@zfsworkstation.peachzone.org/home/taozj/source_code/git/repo/helloworld
    ! [remote rejected] master -> master (n/a (unpacker error))
    error: failed to push some refs to ‘ssh://git@zfsworkstation.peachzone.org/home/taozj/source_code/git/repo/helloworld’

    解决办法：git服务器文件权限问题，调整用户权限，使其具有写权限即可

  • Git Push branch is currently checked out错误

    ! [remote rejected] master -> master (branch is currently checked out)
    error: failed to push some refs to ‘ssh://git@172.29.152.251:/var/www/html/sky/repo/7b4e332c-dfab-11e5-8e1e-70e2840d7ded.git’

    原因：由于git默认拒绝了push操作，需要进行设置
    解决办法：修改.git/config添加如下代码

    [receive]
denyCurrentBranch = ignore

    或者设置全局配置

    git config --system receive.denyCurrentBranch ignore

  • 含有工作目录的远程仓库，push成功后，工作目录的文件没有更新
    解决办法：

    • 每次push后，运行git reset–hard，手动更新版本
    • 使用git的hook，有现成脚本,将脚本保存为post-update，放到远程仓库的./git/hooks目录下，添加可执行权限
