Nginx笔记

nginx作为WebServer对于静态内容有很高的处理效率，并且内置反向代理，负载均衡等功能，是一个很强大的web服务器。一般应用其反向代理将动态内容交给Apache处理，静态内容则由Nginx处理。

安装

依赖

  • 无

安装

  • Ubuntu

    1. 添加nginx源
      在配置文件/etc/apt/sources.list后追加nginx源地址

      deb http://nginx.org/packages/ubuntu/ precise nginx
deb-src http://nginx.org/packages/ubuntu/ precise nginx

    2. 更新系统

      sudo apt-get update

    3. 安装nginx

      sudo apt-get install nginx

使用

基本命令

  • 检测配置文件是否正确

    sudo /usr/sbin/ngnix -t [/etc/ngnix/ngnix.conf]

  • 重载配置文件

    sudo /usr/sbin/ngnix -s reload

应用

添加反向代理记录

  1. 修改nginx配置文件（/etc/ngnix/ngnix.conf），在http配置项中新加server配置项

    http{
    #其他配置
    ......

    server {
        #将域名为jira.wizeye.com.cn的请求全部转发到http://172.29.152.213:8080
        #nginx监听端口，如果设置了`default_server`当nginx找不到虚拟主机时，将请求发送到该虚拟主机
        listen 80 default_server;
        #要转发的域名，可以有多个，空格隔开
        server_name jira.wizeye.com.cn; 
        location / {
            #转发到服务器的IP地址和端口,可以是域名
            proxy_pass http://172.29.152.213:8080;
            #填充Host头部信息为原请求的Host内容，原请求头没有Host则填充server_name第一个域名
            proxy_set_header Host $host;
            #设置`X-Real-IP`头部来标识用户的真实IP
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
    server{
        listen 80;
        server_name stash.wizeye.com.cn;
        location / {
            proxy_pass http://172.29.152.213:7990;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Scheme $scheme;
        }
    }
}
  2. 检测配置文件格式是否正确并重载

配置负载均衡

  1. 修改nginx配置文件（/etc/ngnix/ngnix.conf），添加新的upstream配置项，solr为该负载均衡配置项的名字

    upstream solr{
    server 172.29.152.181:8983;
    server 172.29.152.182:8983;
}

  2. 添加反向代理配置，将转发地址设置为负载均衡名http://solr

    http{
    #其他配置
    server {
            listen 80;
            server_name solr.wizeye.com.cn;
            location / {
                    proxy_pass_header Server;
                    proxy_set_header Host $http_host;
                    #将转发地址设置为`upstream`配置名
                    proxy_pass http://solr;
            }
    }
}
  3. 检测配置文件格式是否正确并重载

错误汇总

  • 出现413 Request Entity Too Large错误
    原因：这个错误一般在上传文件或者git Push时出现，Http的请求体大小超过了nginx限制的最大值
    解决办法：修改配置文件http{}段，将限制设为足够大

    client_max_body_size 2m;
