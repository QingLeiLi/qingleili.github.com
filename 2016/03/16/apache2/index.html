Apache2笔记

Apache作为Web Server三巨头之一，占有很大的市场份额，对于动态内容的解析支持很好，一般与nginx搭配使用。

安装

依赖

  • 无

    安装

  • Ubuntu

    1. 使用apt-get直接安装

      sudo apt-get install apache2
    2. 安装完成后访问127.0.0.1会得到It works!则安装成功

      其他

  • available与enabled
    在Apache2中，mod(功能模块)和site(虚拟主机)的配置都有两个目录，一个是available，一个是enabled，available目录是存放真正的配置内容，但Apache启动的时候只会加载enabled中存在的配置，所以只有在enabled中建立指向available的软连接才可以使功能模块或虚拟主机的配置生效；Apache命令a2enmod和a2ensite就是在available中查找相关配置然后在enabled中建立软连接使其生效。

  • 加载功能模块
    Apache加载的功能模块分为两部分

    1. 编译在内核中的模块，查看命令

      apachectl -l

    2. 配置加载的其他模块，即mods-enabled目录下所有模块

      ls /etc/apache2/mods-enabled

      配置文件

  • /etc/apache2/envvars
    保存了配置文件中使用的各种变量
  • /etc/apache2/apache2.conf
    Apache的主配置文件，设置了大部分全局属性，也可以设置虚拟主机等配置，但虚拟主机配置尽量到site-available中设置

  • 使用

    基本命令

  • apachectl -v
    查看版本相关信息
  • apachectl start(service apache2 start)
    启动服务
  • apachectl stop(service apache2 stop)
    关闭服务
  • apachectl restart(service apache2 restart)
    重启服务
  • apache2ctl configtest(apache2ctl -t)
    检查配置文件
  • service apache2 reload
    重新加载站点
  • a2enmod，a2dismod
    启用和关闭模块
  • a2ensite，a2dissite
    启用和关闭虚拟主机

    其他

  • 查看加载的所有模块

    apachectl -M

应用

搭建git服务器

  1. 确保安装git和Apache，没有的话安装

    sudo apt-get install git-core
sudo apt-get install apache2

  2. 配置Apache虚拟主机，在site-available创建新文件git，并配置虚拟主机

    • 读写仓库都不需要验证身份

      <VirtualHost *:80>
    # 你的域名或IP
    ServerName 127.0.0.1
    # 配置仓库根目录
    SetEnv GIT_PROJECT_ROOT /var/www/repo
    # 可以设置为任意值
    SetEnv GIT_HTTP_EXPORT_ALL 1
    SetEnv REMOTE_USER=$REDIRECT_REMOTE_USER
    # 将/git的请求转发到git-http-backend处理
    ScriptAlias /git /usr/lib/git-core/git-http-backend/
    # 配置目录权限
    <Directory "/usr/lib/git-core*">
       Options ExecCGI Indexes
       Order allow,deny
       Allow from all
    </Directory>
</VirtualHost>

* 读仓库不需要验证身份,写仓库需要

        <VirtualHost *:80>
            # 你的域名或IP
            ServerName 127.0.0.1
            # 配置仓库根目录
            SetEnv GIT_PROJECT_ROOT /var/www/repo
            # 可以设置为任意值
            SetEnv GIT_HTTP_EXPORT_ALL 1
            SetEnv REMOTE_USER=$REDIRECT_REMOTE_USER
            # 将/git的请求转发到git-http-backend处理
            ScriptAlias /git /usr/lib/git-core/git-http-backend/
            # 配置目录权限
            <Directory "/usr/lib/git-core*">
               Options ExecCGI Indexes
               Order allow,deny
               Allow from all
            </Directory>
            <LocationMatch "^/git/.*/git-receive-pack$">
                # 使用了Apache的basic认证，详见Apache
                AuthType Basic
                AuthName "Git Access"
                Require group committers
            </LocationMatch>
        </VirtualHost>

  1. 确保启用Apache的cgid模块，这是解析git-http-backend必须的模块

    sudo a2enmod cgid
  2. 在/var/www/repo下创建仓库test，可以使用http://127.0.0.1/git/test访问到。

    部署Django项目

  3. 安装wsgi模块，并启用

    sudo apt-get install libapache2-mod-wsgi
sudo a2enmod wsgi

  4. 创建wsgi脚本来处理请求，高版本django已经自动生成

    import os,sys
path = os.path.join(os.path.dirname(__file__),'../')
if path not in sys.path:
    sys.path.insert(0, path)
os.environ.setdefault("DJANGO_SETTINGS_MODULE", "lql.settings")

from django.core.wsgi import get_wsgi_application
application = get_wsgi_application()

  5. 在/etc/apache2/site-available中创建lab配置文件，配置虚拟主机

    <VirtualHost *:8080>
    DocumentRoot /home/lql/lab_web_site
    <Directory /home/lql/lab_web_site>
        #是否允许目录浏览，-表示不允许
        Options -Indexes FollowSymLinks MultiViews
        AllowOverride None
        Order allow,deny
        allow from all
    </Directory>
    # 设置处理脚本
    WSGIScriptAlias / /var/www/lab_web_site/wsgi.py
    # 添加静态内容映射，django在非debug模式是不处理静态文件的
    Alias /static  /home/lql/lab_web_site/static
</VirtualHost>

  6. 监听8080端口
    示例使用了8080端口，但Apache默认是不监听8080端口的，配置/etc/apache2/ports.conf,添加

    Listen 8080

  7. 启用lab并重载配置

    # 启用lab配置文件
sudo a2ensite lab
# 检查配置文件是否正确
sudo apache2ctl -t
# 重新加载配置文件
service apache2 reload

    安装mod_python

    mod_python依赖Apxs，是Apache运行python的功能模块。

  8. 检查Apxs是否安装，没有输出路径则没有安装

    # 检查是否安装Apxs
$which apxs2
/usr/bin/apxs2
# 无输出则安装Apxs
$sudo apt-get install python2.7-dev
$sudo apt-get install apache2-dev
  9. 下载Mod_pythod最新源码，后面标号为版本号，并不是对应的python版本

  10. 编译mod_python
    编译安装完成后，mod_python.so会在/usr/lib/apache2/modules/路径下

    # 查看apxs安装路径
$which apxs2
/usr/bin/apxs2
# python安装路径
$which python
/usr/bin/python
# 进入mod_python源码目录
$cd mod_python
# 设置apxs和python路径
$./configure --with-apxs=/usr/bin/apxs2 --with-python=/usr/bin/python
# make时会报错，详见错误汇总
$make
$sudo make install

  11. 创建加载文件并启用
    在/etc/apache2/mods-available中创建加载文件python.load和配置文件python.conf

    # python.load
# 告知Apache加载哪个功能模块
LoadModule python_module /usr/lib/apache2/modules/mod_python.so

    使用Apache命令启用功能模块

    sudo a2enmod python
# 重启服务
sudo service apache2 restart

    错误汇总

  • mod_python make 出错

    connobject.c: In function ‘_conn_read’:
    connobject.c:142:19: error: request for member ‘next’ in something not a structure or union
    apxs:Error: Command failed with rc=65536.
    make[1]: [mod_python.so] Error 1
    make[1]: Leaving directory `/home/general/Desktop/mod_python-3.3.1/src’
    make:     [do_dso] Error 2

    解决办法：修改mod_python/src/connobject.c代码,大概142行，将

    !(b == APR_BRIGADE_SENTINEL(b) ||

    改为

    !(b == APR_BRIGADE_SENTINEL(bb) ||

    重新make即可。

  • [搭建git服务器]clone时403错误

    $ git clone http://172.29.152.251:8080/git/test
    Cloning into ‘test’…
    fatal: unable to access ‘http://172.29.152.251:8080/git/test/‘: The requested URL returned error: 403

    原因：Apache2.2与Apache2.4版本的权限设置有较大差异
    Apache2.2

    Order allow,deny
Allow from all

    Apache2.4

    Require all granted

    解决办法：将git配置文件中权限管理部分改为新版本格式。

  • [搭建git服务器]clone时404错误

    $ git clone http://172.29.152.251:8080/git/test
    Cloning into ‘test’…
    fatal: repository ‘http://172.29.152.251:8080/git/test/‘ not found

    Apache日志

    172.29.153.211 - - [18/Mar/2016:03:16:34 +0000] “GET /git/test/info/refs?service=git-upload-pack HTTP/1.1” 404 4793 “-“ “git/2.5.0.windows.1”

    原因：git的git-http-backend是一个cgi脚本，Apache必须开启cgid模块才能使用它，未启用时会报404错误。
    解决办法：启用Apache cgid模块并重启服务

    sudo a2enmod cgid
sudo service apache2 restart
