Centos更换国内源

  1. 下载163源对应版本的centos文件

  2. 备份原文件

    sudo mv /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup

  3. 将下载的文件拷贝到源目录

    sudo cp CentOS6-Base-163.repo /etc/yum.repos.d/

  4. 清除软件包信息缓存并重建缓存

    yum clean all
yum makecache

环境变量

关于Linux的环境变量的一些配置文件：

  • /etc/profile： 此文件为系统的每个用户设置全局环境信息,当用户登录时,该文件被执行。并从/etc/profile.d目录的配置文件中搜集shell的各个配置文件；

  • /etc/bashrc: 为每一个运行bash shell设置全局环境信息。当bash shell被打开时,该文件被执行；

  • ~/.bash_profile: 用户专用的shell配置信息。当用户登录时,该文件仅仅执行一次!默认情况下,他设置一些环境变量,执行用户的.bashrc文件；

  • ~/.bashrc: 该文件包含专用于你的bash shell的bash信息,当登录时以及每次打开新的shell时,该文件都会被执行；

  • ~/.bash_logout: 当每次退出系统(退出bash shell)时,该文件被执行；

  • ~/.bash_profile: 交互式、login 方式进入bash时运行的脚本；一般会调用~/.bashrc；

  • ~/.bashrc： 交互式 non-login 方式进入bash时运行的脚本。

用户的配置文件的优先级高于系统的优先级，用户可以配置自己的shell来覆盖系统设置。

环境变量的查看

  • 使用env查看所有环境变量

    $ env
HOSTNAME=localhost.localdomain
SHELL=/bin/bash
HISTSIZE=1000
USER=lql
...

  • 使用echo查看单个变量

    $ echo $HOME
/home/lql

添加环境变量

export用于设置环境变量，在shell中执行可以达到目的，但当shell关闭时，该环境变量也会消失，仅仅作用于一个shell，具有临时性。
对于~/.bashrc中设置的环境变量，每当一个shell被启动时，该文件都会被执行，对所有的shell有效。在其中设置环境变量就会作用于该用户所有的shell。
一般通过编辑~/.bashrc来配置环境变量，以python路径示例，在~/.bashrc文件中添加：

export PATH="$PATH:/usr/local/python2.7/bin"

即可将python路径/usr/local/python2.7/bin，添加到环境变量。

错误汇总

  • [ubuntu]apt-get遇到公钥错误

    W: GPG error: http://http.debian.net jessie-backports InRelease: The following signatures couldn’t be verified because the public key is not available: NO_PUBKEY 8B48AD6246925553

    解决办法：运行命令

    gpg –keyserver keyserver.ubuntu.com –recv-keys 8B48AD6246925553
    gpg –export –armor “8B48AD6246925553” | apt-key add -

  • [ubuntu]apt-get遇到依赖问题
    解决办法：安装aptitude，与apt-get一样，但是其在处理依赖方面会好一些，可以提供多套解决方案。

    sudo apt-get install aptitude

  • [centos]yumCannot retrieve repository metadata错误

    Error: Cannot retrieve repository metadata (repomd.xml) for repository: google-chrome. Please verify its path and try again

    原因：centos的某个源没有响应，获取不到仓库信息。
    解决办法：在这里看到是google的源出了问题，直接把Google源停用，这里将其重命名使其失效。 

    # 进入yum源目录
$ cd /etc/yum.repos.d/
# 查看google相关源
$ ls | grep google
google-chorme.repo    CentOS-Vault.repo
# 重命名使其失效
$ sudo mv google-chorme.repo google-chorme.repo.bak
$ sudo mv google.repo google.repo.bak

  • 使用在线脚本安装node时，提示脚本权限不够

    sudo curl -sL https://rpm.nodesource.com/setup | bash -

    原因：该命令是使用curl下载脚本后使用bash执行，虽然给予了管理员权限（sudo），但仅仅是作用于了curl，bash执行时已经没有了权限，所以提示权限不够。
    解决办法：添加sudo -E，给脚本管理员权限。

    sudo curl -sL https://rpm.nodesource.com/setup | sudo -E bash -
  • [centos]pip安装完后，command not found
    原因：pip被默认安装在了python目录中，我的环境中是/usr/local/python2.7/bin，而这个目录不在环境变量中，所以找不到命令。
    解决办法：将/usr/local/python2.7/bin加入环境变量，见环境变量一节。

  • [centos]单独pip可以运行，sudo pip提示command not found
    原因：centos为了安全，sudo执行的命令不会在环境变量中查找，而是在/etc/sudoers中设置了secure_path配置项，指定sudo时执行文件的查找路径，pip在我的环境变量中，但是不在secure_path中，sudo提示找不到命令。

    # /etc/sudoers
Defaults    secure_path = /sbin:/bin:/usr/sbin:/usr/bin

    解决办法：设置一个pip的软连接到secure_path中的目录下或者修改secure_path配置项。

    sudo ln -s /usr/local/python2.7/bin/pip /usr/bin/pip
