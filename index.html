Hexo笔记

安装

依赖

  • NodeJs，Npm

安装

  1. 使用npm直接安装
    npm install hexo -g

其他

  • 升级到最新版本
    npm update hexo -g

使用

基本命令

  • 初始化目录，省略工程名则初始化当前目录
    hexo init [folder]

  • 创建博客文章
    hexo new "Hello World"

  • 生成网站
    hexo generate

  • 运行Hexo服务，默认地址为http://localhost:4000
    hexo server

  • 部署网站
    hexo deploy

应用

使用Hexo创建博客

  1. 初始化目录，并切到博客目录 

    hexo init blog
cd blog

  2. 安装项目依赖包 

    npm install

  3. 生成静态网页 

    hexo generate

  4. 运行项目，查看博客效果，默认地址http://localhost:4000 

    hexo server

部署博客到GitPage上

  1. 登陆GitHub创建仓库(因GitHub限制，仓库名必须为用户名.github.com)

  2. 编辑项目目录下_config.yml配置文件，找到deploy配置项并修改

    deploy:
  type: git  #仓库类型
  repository: git@github.com:用户名/用户名.github.com.git  #仓库地址
  branch: master  #仓库分支

    注意：Hexo 3.0之前的版本，需要修改type:github，并略过3操作

  3. 安装deploy git

    npm install hexo-deployer-git --save

    注意：在Hexo 3.0版本后deploy git被分出来了，需要单独安装，之前的版本无须安装。

  4. 将项目部署到GitHub上 

    hexo deploy
  5. 访问http://用户名.github.io/就应该可以看到博客了。

错误汇总

  • 部署网站时找不到github：Deployer not found: github
    解决办法：见部署博客到GitPage上第2，3条

  • 部署时执行hexo deploy命令行没有任何输出，也没有错误提示
    解决办法：在_config.yml文件中，找到deploy:标签，确保每个冒号（type，repository，branch等）后面有空格，否则就会出现上述问题。

  • 访问网页时，文章不显示，后台报错
    ERROR Process failed: _posts/git.md YAMLException: can not read a block mapping entry; a multiline key may not be animplicit key at line 4, column 1
    原因：yaml配置文件对格式要求很严格，文章的头部信息，每一个键后必须有一个空格，否则会报错。
    解决办法：检查文章头部信息格式将

    tags:git

    改为

    tags: git

    注意空格区别。

Xlwt笔记

安装

依赖

  • pip

安装

pip install xlwt

其他

  • 文档

    http://xlwt.readthedocs.org/en/latest/

使用

基本命令

  • 创建一个book对象

    wbk = xlwt.Workbook()

  • 创建一个工作表

    #参数cell_overwrite_ok，单元格内容是否可以被覆写
sheet1 = wbk.add_sheet('sheet 1')
sheet2 = wbk.add_sheet('sheet 2', cell_overwrite_ok=True)

  • 向单元格写入数据

    #前两个参数为单元格位置（x，y）
sheet1.write(0,1,'test text')

  • 设置单元格样式

    #获取样式对象
style = xlwt.XFStyle()
#获取字体样式  
font = xlwt.Font()
#设置字体 
font.name = 'Times New Roman'
#是否加粗  
font.bold = True
#载入设置
style.font = font
#应用
sheet1.write(0, 1, 'some bold Times text', style)

  • 保存文件（会直接覆盖同名文件）

    wbk.save('test.xls')

xlrd笔记

安装

依赖

  • pip

安装

  1. 使用pip直接安装

其他

  • 文档（date文件夹下有离线版本） 

    http://www.lexicon.net/sjmachin/xlrd.html

使用

基本命令

  • 打开文件,获取book对象

    wb = xlrd.open_workbook('1.xls')

  • 查看包含的工作表

    #获取所有表名
sheetNames = wb.sheet_names()

  • 获得工作表对象 

    #使用索引获取
sh = wb.sheet_by_index(0)
#使用表名获取
sh = wb.sheet_by_name(u'Sheet1')

  • 获取行数和列数

    #行总数  
nrows = sheet.nrows    
#列总数 
ncols = sheet.ncols

  • 获取单元格对象

    #获取单元格对象，可以获取单元格值等信息
cellA1 = sh.cell(0,0)

  • 获取单元格值

    #通过位置坐标直接获取值
cellA1Value = sh.cell_value(0,1)
#通过cell对象获取
cellA1Value = cellA1.value

  • 获取值列表

    #参数为第几列，0-第一列
#获得第1行的数据列表
row_data = sheet.row_values(0)
#获得第一列的数据列表 
col_data = sheet.col_values(0)

错误汇总

  • 在注册表中找不到Python
    Python Version 3.4 required which was not found in the registry
    原因：windows平台下，32/64bit第三方库会分别在32/64bit的注册表中查找python路径，而安装的python 64bit只会把安装信息写入32bit的注册表中，当安装64bit的win32com时，在注册表查找不到python的安装信息。

    Python的一些第三方库只到注册表的HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下寻找Python。但是装好的64位Python在HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下建立Python的信息。所以，在cmd输入regedit然后将HKEY_LOCAL_MACHINE\SOFTWARE\Python\PythonCore\2.7\InstallPath路径下的值，复制到HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Python\PythonCore\2.7\InstallPath路径下，当然没有路径的话要新建。

    解决办法：添加相应的python安装信息到注册表，可以手动新建也可以修改date文件夹下的reg文件来一键导入。

  • import win32com出错
    importError：DLL load failed %1 不是有效的Win32应用程序
    原因：win32com的版本位数与python版本位数不对应
    解决办法：cmd执行python，查看python位数，下载对应安装包，再次安装
